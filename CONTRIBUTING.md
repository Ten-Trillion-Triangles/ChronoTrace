# Contributing to ChronoTrace

ChronoTrace is a Kotlin Multiplatform framework for AI-native structured logging. It captures logs, spans, and full frame snapshots (local variable state) so LLMs can reason about code after execution.

## Build

```bash
./gradlew build
```

This builds the JVM, JS, WasmJs, linuxX64, and macosX64 targets of `sdk-kmp`, plus the server and the Kotlin compiler plugins. First build takes ~10 min; subsequent incremental builds are fast.

## Test

```bash
./gradlew :sdk-kmp:jvmTest
./gradlew :sdk-kmp:jsNodeTest
./gradlew :sdk-kmp:wasmJsNodeTest
./gradlew :sdk-kmp:linuxX64Test
./gradlew :sdk-kmp:macosX64Test
./gradlew :chronotrace-kotlin-plugin:test
./gradlew :chronotrace-server:test
cd sdk-ts && npm test
```

## Module map

- `chronotrace-contract/` — single source of truth for cross-language DTOs (Kotlin `@Serializable` types). Generates `sdk-ts/src/generated/contracts.ts` via `./gradlew :chronotrace-contract:generateTypeScriptContracts`. The generator is verified by `./gradlew :chronotrace-contract:verifyTypeScriptContracts`.
- `sdk-kmp/` — Kotlin Multiplatform SDK. Five source sets: `jvm`, `js(IR)+browser+nodejs`, `wasmJs+browser+nodejs`, `linuxX64`, `macosX64`. Each target must implement the same `ChronoRuntimeHooks`, `ChronoContextElement`, and `HttpTransport` contract.
- `chronotrace-kotlin-plugin/` — K2 IR generation extension for JVM. Rewrites `ChronoLogger.*` and `withTrace`/`withSpan` calls to inject local-variable capture. Mirror extensions exist at `chronotrace-kotlin-plugin-js/` and `chronotrace-kotlin-plugin-wasm/`.
- `chronotrace-kotlin-plugin-gradle/` — Gradle plugin that wires the right compiler-plugin JAR into a consumer build.
- `chronotrace-server/` — Ktor 3.x server (Netty engine, port 8080). REST + WebSocket + MCP. 11 MCP tools.
- `sdk-ts/` — TypeScript SDK for Node.js and browser. Vite plugin included.
- `load-test/` — k6 scripts. Match the `IngestBatch` schema; payloads must use `appId/environment/sdkInstanceId/...` (not the legacy `id/timestamp/...`).

## Architectural invariants

1. **Cross-platform parity.** Every KMP target must honor the same logging and stack-capture contract. The JVM target is the reference. When adding a new actual, mirror the JVM pattern.
2. **Plugin ↔ SDK access.** The K2 compiler plugin resolves SDK symbols (`mergeCaptureFields`, `withTraceCaptured`, `withSpanCaptured`) by FQ-name. These are public in `sdk-kmp` for that reason; do not make them `internal` again.
3. **Visitor descent for `suspend fun`.** The K2 IR generation phase runs before the suspend state-machine is lowered. `visitFunction` must recurse into the inner `SuspendLambda.invokeSuspend` body to instrument `suspend fun` content.
4. **Contract generation.** The TypeScript contract is generated from `chronotrace-contract/`. Hand-editing `sdk-ts/src/generated/contracts.ts` is futile; the file is regenerated by `./gradlew :chronotrace-contract:generateTypeScriptContracts` and verified by `./gradlew :chronotrace-contract:verifyTypeScriptContracts`.
5. **Version source of truth.** `gradle.properties` `CHRONOTRACE_VERSION` is the single source of truth. Subproject `build.gradle.kts` and `sdk-ts/package.json` read from it.

## Code review

All changes go through a code review. Each module has a CODEOWNERS entry (see `.github/CODEOWNERS`). At least one maintainer must approve.
